WEEK 1 BLOG

Display Properties | Sept 11, 2020

To create the homepage and horizontal navigation bar for this blog, I explored the CSS display property. This property defines the display behavior of elements. Property values include:

  • inline: Displays elements as inline elements
  • block: Displays elements as block elements
  • inline-block: Displays elements as block elements on the same level as inline elements
  • table: Displays elements as <table> elements
  • table-cell: Displays elements as data in a <table> element
  • ("CSS Display Property", n.d.)

Specifically, to create the horizontal navigation bar ("CSS Horizontal Navigation Bar", n.d.), I used the inline display value. In HTML, the content of the navigation bar starts off as an unordered list of hyperlinks and their associated titles. 

<div class="nav-bar">
    <ul>
        <li class="nav-content"><a href="index.html" title="Home">HOME</a></li>
        <li class="nav-content"><a href="week1.html" title="New Post">NEW POST</a></li>
        <li class="nav-content"><a href="weeklyblog.html" title="Weekly Blogs">WEEKLY BLOGS</a></li>
        <li class="nav-content"><a href="about.html" title="About">ABOUT</a></li>
    </ul>
</div>

In CSS, I set the unordered list's property, list-style-type, to none. This gets rid of the default bullet points associated with unordered lists. I also set the unordered list's padding value to 0, which removes the default indentation that unordered lists have. 

ul {
    list-style-type: none;
    padding: 0;
}

Additionally, I used the inline property value to position all of the items in the unordered list side-by-side. This was applied to the individual <li> elements. By default, unordered lists are block elements, meaning that each element starts on a new line. I also set the <li> elements' padding property to 10px, which maintains some spacing between the titles. 

.nav-content {
    display: inline;
    padding: 10px;
}

Lastly, to center the navigation bar under the header, I targeted the ul tag that's a descendent of the .nav-bar class and set it's text-align property to center. 

.nav-bar ul {
    text-align: center;
}

To create the homepage's content, I used the display's table property value. I used it to mimic a 2x1 table where the image is in the first cell and the title is in the second cell. In HTML, I start off by placing both the image and title in separate <div> tags to act as containers for the content. Each of these containers will act as a table cell. Both of these entities are encapsulated by another <div> tag to act as the actual table. 

<div class="main-container">
    <div id="main-cont-img"><img src="images/laptop.png" alt="laptop"></div>
    <div id="main-cont-title">WEB DEV BLOG
        <p id="main-cont-body">Vivian Dang | Computer Science></p>
    </div>
</div>

In CSS, I set the div of the main-container class to have the display property value of table. This allows the outer container display its content as a table. I also set the background-color to cadetblue and the table to 100%. As shown in the "Homepage of the Website" image, this main-container spans the entire width of the viewport as the laptop image and title contained inside of it. I also set the top and bottom margin to 20px while the left and right margins are set to 0px. This allows for some spacing between the navigation bar and footer. 

.main-container {
    background-color: cadetblue;
    width: 100%;
    margin: 20px 0px;
    display: table;
}

I also targeted the <div< tags that are descendents of the main-container class and set its display property to be table-cell. In this case, the two <div< descendents of the main-container class are the containers of the laptop image and title. This property will allow the two entities to act as cells in a table, creating the 2x1 table display. Additionally, I set the vertical-align property to be middle, which vertically centers the content. 

.main-container > div {
    display: table-cell;
    vertical-align: middle;
}

For the laptop image, in CSS, I set it's width to be 50%, which allows it to take up 50% of its parent size. In this case, its parent is the <div< of the main-container class, which takes up 100%. This specifies that the cells should be even as the image only fills 50% of the entire table. I also set it's text-align property to be center, which horizontally centers the image. 

#main-cont-img {
    width: 50%;
    text-align: center;
}

Lastly, for the title content of the homepage, in CSS, I set the different fonts and spacing for "WEB DEV BLOG" and "Vivian Dang | Computer Science." 

#main-cont-body {
    font-size: 18px;
    color: lightgray;
    margin: 10px 0px;
}
                    
#main-cont-title {
    padding: 40px;
    font-family: 'Playfair Display', serif;
    font-weight: 300;
    font-size: 4em;
}

From applying these properties, I've learned about the importance of containers when it comes to formatting the display of elements. The concept of inheritance is also important since initially, I couldn't figure out the sizing of the laptop image. I knew that I wanted it to be exactly half of the screen but was just guessing the number of pixels to make it until I realized that it's dependent on it's parent <div> tag. Additionally, I was surprised that I can use the text-align property to even center images.
