Manual and Animated Slideshow | Sept 18, 2020

This week, I explored manual and animated slideshows, which can be created using HTML, CSS, and JavaScript. Even though I've never worked with JavaScript before, I've realized that it is very similar to other programming languages such as Java and Python when it comes to defining and implementing its functions or methods. For both manual and animated slideshows, each slide is treated as an element in an array. By using a standard for-loop, the elements in the array can be tranverse through to create a slideshow effect.

To create a manual slideshow of images, in HTML, every image is declared with the <img> tag and then encapsulated by a <div> container. There are variations on the types of buttons that can be used, but every button must be declared using a <button> tag with the attribute onclick set to some function. The function may be given any name, but according to the article, the most efficient way to create a slideshow is to pass an integer as its argument. This is because, as stated, the slides in the slideshow are treated as elements in an array, so they can be access by their index. -1 can be used to indicate going left while 1 can be used to indicate going right in the slideshow. The function can also take in the index of the element (slide) in the array ("W3.CSS Slideshow", n.d.). As shown in other examples I've read about, an <a> tag can also be used instead of a <button>, but still requires the onclick attribute to be set to some function ("How TO - Slideshow", n.d.) .

For example, to create a manual slideshow as exemplified in the photo above, HTML code will be as followed: 

<div class="container">
    <img class="slides" src="example1.jpg">
    <img class="slides" src="example2.jpg">
    <img class="slides" src="example3.jpg">
    <img class="slides" src="example4.jpg">                        
    <button class="left-button" onclick="move(-1)">&#10094;</button>
    <button class="right-button" onclick="move(1)">&#10095;</button> 
</div>
Code from ("W3.CSS Slideshow", n.d.)

As shown in the photo above, since the buttons are located on the image, they are encapsulated by the same <div> tag as the images. The hex code of #10094 and #10095 are treated as entities to create the ❮ and ❯ icons on the buttons. As stated, there are variations on where the buttons can be located and what they look like. The buttons may also be encapsulated by their own <div> container so that they exist outside of the image. The manual slideshow can also be accompanied with a row of buttons with numbers on it to show the user which slide they are on. Many of these variations can be accomplished in CSS using the flex model or display properties. The functions can also be implemented so that users can only access the slides in order or by clicking on whichever slide they please depending on what buttons are available. As described by my comments, two functions are used to create a slideshow effect. The purpose of the two functions is to choose the appropriate index of the slide in the array. Once the index is selected, all of the other slides are made invisible except the selected slide.

Creating an animated slideshow requires similar code in HTML, except there won't be any buttons since the images will change automatically. This is a basic example, but captions and text may also be added in HTML code for both animated and manual slideshows. 

<div class="container">
    <img class="slides fade" src="example1.jpg">
    <img class="slides fade" src="example2.jpg">
    <img class="slides fade" src="example3.jpg">
    <img class="slides fade" src="example4.jpg">                        
</div>

To create the animated slideshow, in CSS, the @keyframe and @-webkit-keyframe animation rules must be set. The -webkit- prefix allows chrome and safari browsers to render the animation ("CSS @keyframes Rule", n.d.). To create a fade animation, CSS code is as follow: 

.fade {
    -webkit-animation-name: fade;
    -webkit-animation-duration: 1.5s;
    animation-name: fade;
    animation-duration: 1.5s;
}
                          
@-webkit-keyframes fade {
    from {opacity: .4} 
    to {opacity: 1}
}
                          
@keyframes fade {
    from {opacity: .4} 
    to {opacity: 1}
}
Code from ("How TO - Slideshow", n.d.)
