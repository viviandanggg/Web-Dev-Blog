Contact Form | Sept 25, 2020

Creating a contact form that sends directly to a user's email usually involves PHP. This week, I learned about how to create a contact form using just HTML, CSS, JavaScript. This methods uses the mailto: URI scheme. This is accomplished by using <a> tags with the href attribute being set to "mailto:____". This will be written in HTML as followed: 

<a href="mailto:name@example.com">MESSAGE</a>
<a href="mailto:name@example.com?subject=Test Subject">SUBJECT</a>
<a href="mailto:name@example.com?body=Message Body">MESSAGE BODY</a>
<a href="mailto:name@example.com?cc=cc@example.com">CC</a>
<a href="mailto:name@example.com?bcc=bcc@example.com">BCC</a>

The actual form can be written in HTML by using the <form> tag with the name attribute set to "contact" and the method attribute set to "post". However many fields can be added using <label>, <input>, and <textarea> tags. 

<form name="contact" method="post">
    <label>Name</label>
    <input type="text" name="name"/>
    <label>Subject</label>
    <input type="text" name="subject"/>
    <label>Email</label>
    <input type="text" name="email"/>
    <label>Message</label>
    <textarea name="message"/></textarea>
    <input type="submit" name="send" value="Send Message"/>
</form>

CSS can also be used to configure the sizing, margin, and padding of the contact form. JavaScript is more significant in this implementation as it was be used to parse form. Based on the article I read, JavaScript code can be implemented as followed: 

var form = document.forms["contact"];
form.addEventListener("submit", contact_submit, false);

This segment of code targets the contact form and calls an event listener on it. When the user presses "submit," the contact_submit funciton is called. The contact_submit function will be implemented as such: 

function contact_submit(e) {
    e.preventDefault(); // Prevents form from submitting

    // Declares and initalizes variables
    var target = e.target || e.srcElement;
    var to = "someone@example.com";
    var uri = "mailto:" + to;
    var body = "";

    // Set form values to variables
    var name = target.elements["name"].value;
    var subject = target.elements["subject"].value;
    var email = target.elements["email"].value;
    var message = target.elements["message"].value;

    // Builds the body message
    body += message + "\r\n\r\n";
    body += "Name: " + name + "\r\n";
    body += "Email: " + email + "\r\n";

    // Builds mailto URI
    uri += "?subject" + encodeURIComponent(subject);
    uri += "&body=" + encodeURIComponent(body);

    // Opens Mailto in a new Window or tab
    window.open(uri, "_blank";)
}

As described by the comments, the function builds the message and encodes the mailto URI. It listens for the event of the user pressing the send button. This example shows concatenation, which is similar to how Java concatenates Strings. It starts off by grabbing the values from the label and textarea. According to the author of the article, using PHP is a more ideal way to send an email from a website. In order for this way to work, the user needs a mail client setup. If the user uses this method, it is also better if they use JavaScript or some other encoding to encode their email address and then decode it. This is important since it prevent bots from spamming the user's email ("Website Contact Form without PHP.", 2014). Even though using PHP is more ideal, since we are mainly learning JavaScript, CSS, and HTML in this course, I thought that this was an interesting alternative. If anything, this example uses the <contact>, <input>, <label>, and <textarea> tags which are useful for obtaining some sort of user input.
