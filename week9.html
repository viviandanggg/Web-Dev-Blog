React JSX, Props, and Components | Nov 20, 2020

As we learn more about React, the idea of props, components, and JSX still kind of confuses me, so this week, I explored documentation that further explain these parts of React. These pages of documentation comes from the official React website. It explains these concepts by providing examples and showing how the concepts interact with each other.

Firstly, we'll explore JSX. The documentation describes JSX as a syntax extension to JavaScript. It is recommended to be used with React to determine what the UI should look like. JSX produces React elements. The documentation provides the following example. 

const name = 'Josh Perez';
const element = <h1>Hello, {name}</h1>;

function formatName(user) {
    return user.firstName + ' ' + user.lastName;
  }
  
  const user = {
    firstName: 'Harper',
    lastName: 'Perez'
  };
  
  const element = (
    <h1>
      Hello, {formatName(user)}!
    </h1>
  );
  
ReactDOM.render(
  element,
  document.getElementById('root')
);

As exemplified by the {name} and {formatName(user)}, any valid JavaScript expressions can go inside of curly braces. Valid expressions include functions, object properties, and variables.

The documentation then describes JSX as an expression as well. After compilation, JSX expressions become regular JavaScript objects and function calls. This allows developers to assign JSX to variables, accept it as arguments, return it from functions, and use it inside of if statements or for loops.

The documentation then talks about specifying attributes with JSX. Quotes can be used to specify string literals as attributes. Curly braces can also be used to specify attributes that are expressions. Only either quotes or curly braces can be used for one attribute, never both. 

// Quotes                        
const element = <div tabIndex="0"></div>;

// Curly Braces 
const element = <img src={user.avatarUrl}></img>;

Additionally, if a tag is empty, it can just be immediately closed with />. Furthermore, JSX tags may contain children as demonstrated by the following example. 

const element = (
  <div>
    <h1>Hello!</h1>
    <h2>Good to see you here.</h2>
  </div>
);

Lastly, the documentation explains that JSX represents objects called React elements. They essentially represent what is displayed on the screen. The two following blocks of code represent the same thing. 

const element = (
  <h1 className="greeting">
    Hello, world!
  </h1>
);

const element = React.createElement(
  'h1',
  {className: 'greeting'},
  'Hello, world!'
);

We'll now explore React components and props. The documentation states that components allow you to split up the UI into independent, reusable pieces. It states that "components are conceptually JavaScript functions that accept arbitrary inputs called props and return React elements that describe what should appear on the screen." This means that we can make components by writing JavaScript functions as exemplified below. 

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>
}

These components can then be rendered by doing the following. 

const element = <Welcome name="Sara" />;

Components can also consist of other components. React apps usually have a top-level App component that can consist of other components as demonstrated. 

function App() {
  return (
    <div>
      <Welcome name="Sara" />
      <Welcome name="Cahal" />
      <Welcome name="Edite" />
    </div>
  );
}

Lastly, the documentation describes how props are read only. Components should never modify its own props.
